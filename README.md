# DevXP ⚡

自分が書いたコードやコミットなど開発の記録・経験が**経験値（XP）**となり、自分の将来的な活動を解放する原動力となる。
泥中を進むような開発体験をゲームのような可視化された体験へと変え、後に振り返るとセーブデータや進捗が残っており、自分がどういう道を歩んできたかを振り返ることのできる**開発体験ジャーナルアプリ**。

## コンセプト

これまでの経験値系アプリとの違いは、**貯めた XP を使って自分への報酬をアンロックする**点にある。コーヒー一杯・散歩・好きなものを買う、といった報酬に必要 XP を設定し、コーディングで稼いだ XP で解放していく。

運営側が何かを提供するのではなく、**ユーザー自身が自分の行動に意味付けを行う**——過程に価値を見出すことがコンセプトの核心。

## アーキテクチャ

```
[ローカルの git リポジトリ]
        ↓  post-commit hook / devxp sync
[CLI: devxp (npm)]
        ↓  diff 送信（API キー認証）
[Web API: Next.js / Vercel]
        ↓  保存
[DB: Supabase]
        ↓  表示
[Web ダッシュボード]
```

CLI は git のコミット差分（追加/削除行数）を収集して Web API へ送信するのみを担う。XP の算出ロジックは Web API 側で非公開のまま処理する——CLI は npm で OSS 公開されるため、換算ロジックをそちら側に置くと改ざん（チート）が容易になってしまうことへの対策である。

## XP 換算ルール

| アクション | XP |
|-----------|-----|
| 追加 1 行  | +2 XP |
| 削除 1 行  | +1 XP |

> PR・コミット単位で集計し、プライベート/パブリック問わずローカルの `git log` から収集する。

## CLI の使い方

```bash
# インストール
npm install -g devxp

# 初期設定（Web アプリで発行した API キーを登録）
devxp init

# 手動で XP を同期
devxp sync

# post-commit hook を自動設定（コミットのたびに自動送信）
devxp install
```

## 主な機能（MVP）

- **ダッシュボード**：レベル / XP バー / 利用可能 XP / 今日の行数
- **アクティビティヒートマップ**：GitHub の草グラフを超えた、コーディング量の可視化
- **ご褒美システム**：自分で報酬を定義し、XP で解放する
- **バッジ**：コーディング実績のアンロック
- **ジャーナル**：振り返り——自分の開発の道のりを「セーブデータ」として残す

## ロードマップ
